## Architecture Decisions Made

### 1. VerificationStatus as Enum Cast
The `verified` column on Photo is now cast to a `VerificationStatus` BackedEnum. This means:
- `$photo->verified` returns an enum instance, not an int
- Use `->value` for ordering comparisons (`>=`, `<`)
- Use direct enum identity (`===`) for equality
- Laravel auto-extracts `->value` in Eloquent `where()` clauses
- `json_encode()` serializes to the backing int value

### 2. `geom` Column Hidden from JSON
The `geom` column contains binary spatial data (`ST_GeomFromText(...)`) that cannot be UTF-8 encoded. Added to Photo `$hidden` to prevent JSON serialization errors in all API responses.

### 3. Team Type Resolution via Relationship
`Team::getTypeNameAttribute()` queries `teamType()` relationship instead of hardcoding ID-to-name mappings. This ensures correct behavior with `RefreshDatabase` in tests (where IDs depend on insertion order).

### 4. School Approval as MetricsService Trigger
For school teams, `TeamPhotosController::approve()` is the point where `TagsVerifiedByAdmin` fires — not `AddTagsToPhotoAction`. This means teacher approval IS the verification event. Summary/XP must be generated at tagging time regardless of trust level, or they'll be null at approval time and MetricsService will extract zero.

### 5. Tag Editing Resolves Strings to FKs
`TeamPhotosController::updateTags()` accepts category/object as string keys (`smoking`, `cigarette_butt`) and resolves them to `category_id`/`litter_object_id` via `Category::where('key', ...)` and `LitterObject::where('key', ...)`. This matches the API contract expected by the frontend.

### P0 Fix: Soft Deletes + Metrics Reversal (8 files)

School teams will delete bad photos from day one. The delete controllers hard-deleted the photo row before metrics could be reversed. The `DeletePhotoMetrics` listener was broken — `ImageDeleted` event has no `photo_id` property, so the listener always no-oped.

**Fix:** Added SoftDeletes so the row persists, call `MetricsService::deletePhoto()` synchronously in controllers before soft-deleting.
