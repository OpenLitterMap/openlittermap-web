## Session 7: Leaderboard Correctness Fixes

Audited and fixed 4 correctness issues in the leaderboard system. Added 3 new tests (12 total).

### FIX 1: Daily `bucket_date` index coverage

Controller code already filtered by `bucket_date` for daily queries, but `idx_leaderboard` index was missing `bucket_date`. Created migration `2026_02_24_160843` to rebuild the index:
```sql
CREATE INDEX idx_leaderboard ON metrics (timescale, location_type, location_id, year, month, bucket_date, xp DESC)
```

### FIX 2: Zero-XP pruning in Redis

After a delete, `ZINCRBY` could leave ghost entries with score â‰¤ 0. Added `ZREMRANGEBYSCORE {scope}:lb:xp -inf 0` after delete's ZINCRBY in `RedisMetricsCollector`. Works inside the pipeline because Redis executes pipeline commands sequentially on the server.

### FIX 3: Deterministic tie-breaking

Added `->orderBy('user_id')` after `->orderByDesc('xp')` in time-filtered leaderboard queries. Ensures consistent pagination when multiple users have the same XP.

`getCurrentUserRank()` already adds 1 to `ZREVRANK` (0-indexed â†’ 1-indexed). Added PHPDoc and inline comment for clarity.
