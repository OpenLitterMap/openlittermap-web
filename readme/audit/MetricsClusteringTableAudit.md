## dirty_teams (3 columns, 2 indexes)

**Rows:** Dynamic (populated by PhotoObserver, consumed by clustering)

All columns active. Same schema as dirty_tiles but for team-level reclustering.

---

## Notes

- `metrics` table is the SINGLE source of truth for all aggregated stats. Written exclusively by `MetricsService`.
- `clusters` table is fully regenerated by clustering commands ‚Äî not incrementally updated.
- `dirty_tiles` and `dirty_teams` are queue-like tables consumed by clustering commands.
- No FK constraints on any of these tables (by design ‚Äî they're derived data).
