# 2026-03-24

## v5.1.0 — Enriched auth response + profile performance optimization
- `POST /api/auth/token` and `POST /api/auth/register` now return full profile data (user, stats, level, rank, team) — collapses mobile login from 3-4 API calls to 1
- Extracted `ResolvesUserProfile` trait shared by AuthTokenController, RegisterController, and ProfileController
- Removed achievements and global_stats from profile responses (unused by frontend)
- Split profile into lean mobile response (~100ms) and full SPA response (+ streak, locations, percentages)
- Bumped `/api/auth/token` rate limit from 5/min to 10/min for Apple review safety
- Cleaned up profile Pinia store (removed dead global_stats/achievements state)

## v5.1.1 — Fix 40s profile load on production
- Added composite index `photos_user_location_counts_idx` on `(user_id, deleted_at, country_id, state_id, city_id)` — fixes 40s COUNT DISTINCT query
- Increased location counts cache TTL from 5min to 1hr (keyed by upload count for auto-invalidation)
- Fixed duplicate `resolveUserStatsLight` call in profile/index (was running metrics query twice)
- Removed temporary timing instrumentation from profile endpoint
- Production fix: `CACHE_DRIVER` must be `redis` (not `file`) — file cache caused 24s+ response times
