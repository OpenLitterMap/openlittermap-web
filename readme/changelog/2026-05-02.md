# 2026-05-02

## v5.10.0 ‚Äî Configurable CSV export formats: split (v5) / joined (v4-style)
- Add `format` query param to user, team, and location CSV exports. Accepts `split`, `joined`, or `split,joined` (comma-separated)
- `split` (default): current v5 layout ‚Äî object/type/material in separate columns
- `joined`: v4-style ‚Äî one column per `{type}_{object}` (e.g. `spirits_bottle`, `beer_can`, `water_bottle`); bare `{object}` when no type. Suppresses split block + TYPES block. MATERIALS / brands / custom blocks still emitted
- `split,joined`: both blocks emitted, split first then joined. MATERIALS appears once
- Add Split/Joined checkboxes next to Export CSV on `/uploads`, the Teams Photos tab, and the Locations download page; Export disabled when both unchecked. Default Split-only
- Wire up `Locations/Location/Controls/Download.vue` to actually call `POST /api/download` (was a stub `alert('todo')`)
- Backend: `CreateCSVExport::normalizeFormats()` deduplicates + lowercases + trims; empty/garbage falls back to `['split']`. Threaded through `ProfileController::download`, `TeamsController::download` ‚Üí `DownloadTeamDataAction`, `DownloadControllerNew::index`
- Add `tests/Feature/Exports/CreateCSVExportFormatTest.php` ‚Äî 7 tests covering split-only, joined-only, both, empty defaulting, no-type bare-object fallback, per-category disambiguation, and non-collapsing of materials into joined columns
