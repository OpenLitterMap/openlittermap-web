# 2026-05-03

## v5.11.0 — Long-format CSV export ("Full-detail")
- Add `layout` query/body param (`wide` default | `long`) to user, team, and location CSV exports. Long emits one row per tag dimension instead of one row per photo
- Long-format schema (14 fixed columns): `photo_id, datetime, lat, lng, team, verification, category, object, type, material, brand, custom_tag, quantity, photo_tag_id`
- Per-extra row emission (NOT cartesian): an object PhotoTag with N materials and M brands emits `1 + N + M` rows (bare object + one per material + one per brand). `photo_tag_id` column lets users dedupe via `GROUP BY` before summing — naive `SUM(quantity)` overcounts by design
- Brand rows use brand-specific qty from the `photo_tag_extra_tags` pivot; material/object rows use parent PhotoTag qty; custom_tag rows use qty=1
- Brand-only / material-only / custom-only PhotoTags emit only their extra rows (no bare-object row)
- Photos with no PhotoTags emit zero rows in long format (long-mode `query()` adds `whereNotNull('summary')` to skip per-row hydration)
- Frontend: radio (Number-based / Full-detail) with nested Separate/Combined checkboxes that disable when Full-detail is selected. Visible description paragraphs under each option + `floating-vue` ⓘ tooltips for depth. Wired into `UploadsHeader.vue`, `TeamPhotosHeader.vue`, `Locations/Location/Controls/Download.vue`. `MyTeams.vue` + `TeamSettingsTab.vue` get split icon buttons (Number-based + Full-detail)
- `username` column intentionally omitted from v1 long schema (privacy posture for off-platform CSVs)
- Filename suffix `_number-based_` / `_full-detail_` added to all three exports via `CreateCSVExport::layoutSlug()` static helper
- Backend: `parseLayout(?string)` / `normalizeLayout(?string)` / `layoutSlug(string)` static helpers on `CreateCSVExport`; new `headingsLong()` / `mapLong()` / `longRow()` private methods; `headings()` and `map()` dispatch on `$this->layout`. Constructor early-returns and skips the wide pre-scan in long mode (saves ~6 queries per dispatch). `query()` eager-loads `team:id,name` only in long mode
- 13 new tests in `tests/Feature/Exports/CreateCSVExportLongFormatTest.php` (70 export tests passing total). Real-data verification: 83-tag champion photo emits 112 rows in 1.5ms; 56-brand outlier emits 56 rows
- Diagnostic: `Log::info('team-export-request', ...)` in `APITeamsController::download` logs `raw_layout`/`parsed_layout`/`raw_format`/`parsed_formats`. Helps diagnose stale Horizon worker issues (which require `php artisan queue:restart` after deploy)
- Update `readme/ExportData.md` with the Wide vs Long Layout section, 14-column schema table, row-emission rules, worked example, and the API↔UI naming note ("API uses `wide`/`long`, UI shows Number-based/Full-detail")
