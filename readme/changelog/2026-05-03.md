## v5.10.1 — CSV export perf + Download.vue UX fix
- Fix `Download.vue` wiping the email field when an unauth'd location download fails — reset now only happens on success
- Collapse three filtered `photo_tags` scans in `CreateCSVExport::__construct` into one when `format=joined` is requested (constructor was scanning the same scope for active objects, active types, and joined triples — now derives all three from a single triple result)
- Drop three lookup queries from `CreateCSVExport::buildJoinedColumns` (Category, LitterObject, LitterObjectType) — keys now built from already-loaded `$this->categoryObjects` / `$this->types` instead of re-querying
- Gate per-tag `joinedLookup` write on `$this->emitJoined` in the `map()` hot path so split-only exports skip the work
- Add `CreateCSVExport::parseFormats(?string)` static helper; `ProfileController::download`, `TeamsController::download`, `DownloadControllerNew::index` now call it instead of inlining `array_filter(explode(...))`
- Open `TeamPhotosController::memberStats` to all team members (was leader/school_manager only) so the Members tab renders for non-leaders. Pseudonyms still apply on `safeguarding=true` (school) teams; non-school visibility is intentional and now documented inline
- Fix CSV `total_tags` column reading `summary.totals.litter` (objects-only) instead of the `photos.total_tags` DB column (objects + materials + brands + custom). Custom-only / brand-only / material-only photos previously showed 0
- Add `format` query param to user/team/location CSV exports: `split` (v5 default), `joined` (v4-style `{type}_{object}` columns, suppresses split + TYPES blocks, keeps MATERIALS), or `split,joined` for both.
