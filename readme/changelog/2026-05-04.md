# 2026-05-04

## v5.11.3 — Collapsible export drawer, plain-language format options
- New shared `ExportDrawer.vue` (light + dark theme) replaces the inline radio + nested-checkbox format selector that was duplicated across `UploadsHeader.vue`, `TeamPhotosHeader.vue`, and `Locations/Location/Controls/Download.vue`. The Export CSV button on each page now toggles the drawer open/closed below the filter row (or below the Download button on the location page)
- Three flat radio options replace the old "Number-based + Separate/Combined checkboxes / Full-detail" matrix: "For Excel or Google Sheets" (wide+split), "For analysis tools (pandas, SQL, R, Tableau)" (long), "For legacy v4 scripts" (wide+joined). Default = Excel
- Confirmation state lives inside the drawer: "Queued — we'll email it to {email} when it's ready." Dismissable via Cancel or by toggling the Export button again
- The previous "tick both Separate and Combined" power-user combo is gone from the UI but the API still accepts `format=split,joined` directly
- Backend, Pinia store actions, filename generation, and request payloads are untouched — the drawer is presentational only

## v5.11.2 — Location export now auth-only
- `POST /api/download` is now behind `auth:sanctum` (moved to its own Downloads block in `routes/api.php`, out of the user-routes group). Anonymous CSV exports were an abuse vector against the queue + S3 + outbound mail; tightening the route to authenticated users only is the right baseline. Guests previously supplied an email field; that path is gone, the email validation + random `_g{Str::random(6)}` suffix branch in `DownloadControllerNew` is gone, and `Download.vue` now shows a "sign in to download" prompt for unauthenticated visitors instead of an email input
- Throttle simplifies: `csv-export` keys on `user_id` (with IP fallback as a defensive backstop) — no more per-`(ip,email)` pair, no more per-IP `perHour(20)`. All export endpoints are auth-only now so the per-account semantic is correct
- Test coverage: drop the two guest-validation tests, add `test_guest_request_is_rejected_with_401` against the new contract

## v5.11.1 — CSV export audit follow-ups
- Pseudonymize `TeamPhotosController::memberStats` on `$team->isSchool()` instead of the `safeguarding` flag — closes a privacy regression where any school team with `safeguarding=false` (DB default) would expose minor classmates' real names + per-student stats to peers via the Members tab
- Pin one `$now` per export request in `DownloadTeamDataAction`, `ProfileController::download`, and `DownloadControllerNew::index` so timestamp / `Y-m-d_His` / `Y/m/d` segments of the S3 path can't disagree across a second boundary
- Guest exports now use a random suffix (`_g{Str::random(6)}`) instead of the shared `_uguest` so two simultaneous guest downloads of the same location can't overwrite each other
- `Str::slug()` DB-sourced location names before they land in the S3 key — defends against `..` or punctuation in country/city names producing malformed keys. Falls back to `{locationType}-{id}` when slug is empty (all-non-Latin names like 中国 / العربية / Ελλάδα), so the file stays distinguishable in S3 listings
- Add a per-IP `perHour(20)` limit alongside the existing per-(user|ip+email) `perMinute(3)` limit on `csv-export`. Stops a single IP spraying victim emails to multiply its budget under the per-address limit
- Trim wide-mode export query SELECT to the 12 columns `map()`/`headings()` actually read; trim long mode to 9. Skips `geom` BLOB and other unused columns — cuts hydration on large country exports. `updated_at` kept in both for Eloquent timestamp-accessor safety
- Skip the `extraTag` eager-load JOIN in wide mode when `hasCustomTags=false`
- Frontend `429` UX in `Download.vue`, `UploadsHeader.vue`, and `TeamPhotoList.vue`: show *"Too many exports — try again in {Retry-After}s."* instead of the generic *"Export failed."* when the throttle fires
- Add ⚠ warning to `readme/ExportData.md` explaining long-format `SUM(quantity)` overcounts and how to dedupe
- Update memberStats docblock + `CreateCSVExport::query()` comment to match the new behaviour after fixes
