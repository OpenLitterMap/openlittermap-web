# 2026-06-02

## Session: Allow new users to create one team on signup

- v5.11.4 — `RegisterController::create()` now sets `remaining_teams => 1` for new users, allowing each new signup to create one (non-school) community team. School teams remain gated behind the `school_manager` role; existing users and the DB column default (0) are unaffected. Added `CreateNewUserTest::test_new_user_can_create_one_team_on_signup`.

## Session: Fix community team creation not appearing on the live map

- v5.11.4 — Fixed `TeamCreated` broadcasts never reaching the global map live feed. `LiveEvents.vue` listened for `TeamCreated` on the `main` channel, but the event broadcasts on the `teams` channel under the `team.created` alias — a triple mismatch (channel, event name, and payload key). Moved the listener to `Echo.channel('teams').listen('.team.created', …)` with matching `leaveChannel('teams')` cleanup, and fixed `Notifications/TeamCreated.vue` to read `payload.team_name` (was `payload.teamName`). School teams remain on their private channel and are intentionally excluded from the public feed. Added `TeamsTest` broadcast-contract tests for community (public `teams` channel + name) and school (private channel, no name) teams.
