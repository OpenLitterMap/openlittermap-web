# Changelog — 2026-06-13

## Session: Fix `/global` map crash — glify.longitudeFirst is not a function

- Fixed `TypeError: glify.longitudeFirst is not a function` on `/global` (`glifyHelpers.js:25`). Root cause: the Vite 6→8 upgrade (commit df704060) brought a newer esbuild dep-optimizer that pre-bundles the UMD `leaflet.glify` as `export default require_glify_browser()` — exposing the module **namespace** (`{ Glify, glify, default }`) as the default export instead of the `Glify` instance. So `import glify from 'leaflet.glify'` bound `glify` to the namespace, which has no `longitudeFirst` (it lives on the instance at `.glify`/`.default`). The `leaflet.glify` package itself was unchanged (3.3.1 before and after). (v5.12.4)
- `glifyHelpers.js`: import as `glifyModule` and normalize with `glifyModule.glify ?? glifyModule.default ?? glifyModule` — resolves to the instance across all bundler interop shapes (dev esbuild namespace, prod Rollup `.default`/instance). One-line, no dependency or Vite config changes. (v5.12.4)
- Verified end-to-end with a puppeteer load of `https://olm.test/global`: glify pageerror present before the fix, gone after (HMR). (v5.12.4)
- Fixed a second, latent `/global` crash exposed by the first fix: `TypeError: Cannot read properties of null (reading 'containerPointToLayerPoint')` on map pan (and popup auto-pan). Root cause: glify's `CanvasOverlay` subscribes to map `moveend`/`resize`/`zoomanim` using a shared-prototype `_reset` handler. The glify manager tracks `currentInstance.layer`, but the overlay actually attached to the map can be a different instance, so `removeCurrentInstance()` never unsubscribes the live overlay; Leaflet then nulls the detached overlay's `_map`, and its `_reset` throws on the next `moveend`. Each points reload orphaned another listener (verified: moveend listeners grew 5→8 over 4 pans). Previously masked because points never rendered before the import fix. (v5.12.4)
- `glifyHelpers.js`: added `purgeDetachedOverlayListeners(map)` — after each removal it sweeps `map._events` and calls the public `map.off(type, fn, ctx)` with each listener's exact stored `fn`/`ctx` (the only call that reliably matches) for any glify overlay whose `_map` is already null. `removeCurrentInstance()` captures the live map before glify nulls `_map`, then invokes the sweep. Verified: across 6 pans the glify moveend listener count stays flat at 1 (0 detached) and 0 `containerPointToLayerPoint` errors. (v5.12.4)
