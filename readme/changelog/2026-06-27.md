## Session: Subscribe endpoint validation (mass-email malformed address fix)

- Restored the mailing-list subscribe entry point that the live `Footer.vue` form still posts to (`POST /subscribe` had been removed, so every real subscribe attempt was 404ing). Recreated `SubscribersController` (invokable) + the `web.php` route, now with strict server-side validation: `required|max:100|email|regex:/@[^@\s]+\.[a-z]{2,}$/i|unique:subscribers`. The dotted-domain regex rejects RFC-valid-but-undeliverable single-label domains like `Estherbarriga@8` (the address that triggered the SES `Invalid domain name: '8'` failure during the Update 28 mass send) — Laravel's default `email` rule alone accepts these. Added 7 tests to `EmailSubscriptionTest.php` (valid insert, dotless-domain rejection across 6 real malformed samples, multi-level/uppercase domain acceptance, duplicate + missing-email rejection). 19 tests passing. (v5.12.7)

## Session: Email deliverability Phase 1 (ledger, suppressions, SNS webhook, backfill)

- **Three new tables** (`email_suppressions`, `email_events`, `email_sends`) + `users.email_verified_at`. `email_sends` is the campaign ledger — `uniq(campaign, email)` is the atomic claim lock, making a bulk send runnable, interruptible, resumable, and auditable. (Prod note: the composite unique on utf8mb4 needs `innodb_large_prefix` + DYNAMIC row format on MySQL 5.7 — confirm prod version before migrating.)
- **`App\Support\EmailAddress::isSendable()`** — single source of truth for deliverability (filter_var + dotted-domain check). Refactored the inline regex out of `SubscribersController` into it; controller + send command now share one check.
- **`olm:send-email-to-subscribed` rewritten** around the ledger: requires `--campaign`, atomic claim, re-run-safe (skips `accepted`/in-flight `queued`, re-evaluates previously-skipped rows so an un-suppressed/fixed address sends on rerun), suppression + invalid + duplicate guards, dry-run report that writes nothing. New switches: `--limit`, `--retry-failed`, `--retry-stale-queued=N`; `--test=` renamed to `--only-email=` (breaking flag change); `--chunk` retained. Ledger `accepted`/`failed` written in the job's terminal hooks (`handle()`/`failed()`), never on first throw.
- **`EmailRecipient` DTO** — `DispatchEmail` no longer assumes a `User`; carries the correct `sub_token` for users and subscribers alike.
- **SNS webhook** `POST /webhooks/aws/ses/sns` (`SesSnsWebhookController`): AWS signature verification (`aws/aws-php-sns-message-validator`, container-injected for testability) + TopicArn check before anything else, reads the raw `text/plain` body, CSRF-exempt (`webhooks/*`). Permanent bounce → suppress, Transient → ignore, Undetermined → log only, Complaint → suppress (complaint outranks bounce). Idempotent on `(sns_id, email)`. New env `SES_SNS_TOPIC_ARN` → `services.ses.topic_arn`.
- **`email:import-suppressions {file}`** — backfills the SES account-level suppression list (`BOUNCE→bounced`, `COMPLAINT→complained`, source=backfill, complaint-preserving, idempotent).
- **§9b email verification**: added `users.email_verified_at`; `User::confirmEmail()` now dual-writes it alongside the legacy `verified` boolean. `email:backfill-verified-at` (dry-run by default, `--apply`) sets `email_verified_at = created_at` for existing verified users. Send eligibility unchanged (`emailsub=1`, includes `verified=0`).
- Added `aws/aws-php-sns-message-validator` (signature verification only). Tests: `tests/Feature/Email/` — EmailValidation, DispatchEmail, EmailSendLedger, EmailUpdate, EmailWebhook, EmailSuppressionImport, EmailVerificationMigration. Full suite: 1335 passed, 1 skipped, 0 failures. (v5.12.8)
