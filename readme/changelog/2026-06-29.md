# 2026-06-29

## Session: Web HEIC upload fix (FilePond gate, EXIF deferral, configurable heif-convert)

- **Version bump → v5.12.13** (patch — bug fix).
- Fix: web HEIC uploads (e.g. iPhone `.HEIC` via `/upload`) were failing silently. Root cause spanned three gates, now all fixed end-to-end. (v5.12.13)
- Frontend: `Upload.vue` — FilePond `acceptedFileTypes` now uses `image/heic`/`image/heif` (MIME) and a new `fileValidateTypeDetectType` resolver maps an empty `file.type` (Chrome/Firefox report `""` for HEIC) to a MIME by extension. Previously the `.heic`/`.heif` extension entries never matched a real File's type, so FilePond rejected HEIC client-side and the upload request never fired. (v5.12.13)
- Backend: `UploadPhotoRequest::after()` — skips the raw `exif_read_data()` GPS/datetime validation for HEIC (PHP can't read HEIC EXIF pre-conversion → returned `false` → false `no_exif` 422). Mirrors the existing `rules()` HEIC special-casing. (v5.12.13)
- Backend: `UploadPhotoController` — web branch now extracts GPS/datetime from the post-conversion EXIF (`heif-convert` embeds GPS/datetime into the JPEG) and returns a typed `no_gps` 422 if genuinely missing. Normalizes `$exif` to `[]`. Resolves the documented web-mode HEIC "known limitation". (v5.12.13)
- Backend: `config/services.php` + `MakeImageAction` — the `heif-convert` binary path is now configurable via `HEIF_CONVERT_PATH` (default `heif-convert`, resolved via PATH for prod/CI). Lets local Valet — whose launchd-started php-fpm has a restricted PATH excluding `/opt/homebrew/bin` — point at the absolute path. Added `HEIF_CONVERT_PATH=/opt/homebrew/bin/heif-convert` to local `.env` (not `.env.testing`). (v5.12.13)
- Frontend: `Upload.vue` — long upload errors (e.g. HEIC conversion failure) now render in a contained, wrapped banner (`uploadError` ref) instead of overflowing the FilePond inline status label off-screen. Inline label kept short. (v5.12.13)
- Tests: +2 web-path tests in `HeicUploadTest` — `test_web_heic_upload_extracts_gps_from_exif_and_succeeds` and `test_web_heic_upload_without_gps_returns_no_gps`. Full upload/validation/lifecycle/HEIC suites green (22 HEIC + 61 upload/validation/lifecycle). (v5.12.13)
- Docs/skills: updated `readme/Upload.md` (web HEIC now supported; configurable binary path) and the `photo-pipeline` skill (corrected stale ImageMagick `convert` references to `heif-convert`; documented EXIF deferral + configurable path). Production `Server dependencies` block pending verification against the live box. (v5.12.13)
